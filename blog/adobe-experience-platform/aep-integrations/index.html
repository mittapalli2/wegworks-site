AEP Integrations Guide: All Supported Connections & Destinations

Adobe Experience Platform is only as powerful as its connections. This comprehensive guide covers every major integration category—from Adobe native apps to advertising platforms, email tools, CRMs, and cloud storage destinations.

Sources vs. Destinations

AEP integrations fall into two categories: Sources (data coming IN to AEP) and Destinations (data going OUT from AEP). Understanding this distinction is fundamental to architecting your data flows.

Adobe Experience Cloud Integrations

AEP serves as the backbone for Adobe's entire Experience Cloud suite. These native integrations offer the deepest functionality and real-time data sharing.

How Adobe Integrations Work

Adobe-to-Adobe integrations leverage the Experience Platform Edge Network for real-time data sharing: 

// Web SDK sends data to Edge Network
alloy("sendEvent", {
  "xdm": {
    "eventType": "web.pageView",
    "web": {
      "webPageDetails": {
        "name": "Product Detail Page",
        "URL": "https://example.com/product/123"
      }
    }
  }
});

// Edge Network simultaneously:
// 1. Writes to AEP Data Lake
// 2. Updates Real-Time Customer Profile
// 3. Sends to Adobe Analytics
// 4. Triggers Adobe Target personalization
// 5. Evaluates Journey Optimizer triggers

Source Connectors (Data Ingestion)

AEP sources bring data INTO the platform. Here are the major categories:

Adobe Sources

  • Adobe Analytics — Historical and streaming data import
  • Adobe Audience Manager — Segment and trait migration
  • Adobe Campaign — Email engagement data
  • Marketo — B2B marketing data and leads

Cloud Storage Sources

  • Amazon S3 — Batch file ingestion
  • Azure Blob — Microsoft cloud data import
  • Google Cloud Storage — GCP data ingestion
  • SFTP — Legacy file transfers

Database Sources

  • Amazon Redshift — Data warehouse queries
  • Google BigQuery — Analytics data import
  • Snowflake — Cloud data platform
  • PostgreSQL / MySQL — Operational databases

Streaming Sources

  • HTTP API — Real-time event streaming
  • Apache Kafka — Event stream ingestion
  • Amazon Kinesis — AWS streaming data
  • Azure Event Hubs — Microsoft streaming

Integration Best Practices

Start with Identity Strategy

Define your identity namespaces before connecting sources and destinations. Poor planning leads to fragmented profiles.

Use Streaming Where Possible

Streaming integrations update profiles in real-time. Batch introduces latency. Design for streaming-first.

Implement Governance First

Apply data labels BEFORE connecting destinations. It's much harder to retrofit governance after data is flowing.

Monitor Integration Health

Track ingestion success rates, profile update latency, destination errors, and identity graph quality.

Plan for Scale

AEP pricing is based on data volume (profiles, events). Estimate growth before enabling high-volume integrations. Start with highest-value data sources and expand systematically.
