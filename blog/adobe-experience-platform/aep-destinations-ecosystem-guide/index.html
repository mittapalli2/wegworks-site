AEP Destinations Explained: How CJA, Journey Optimizer, Analytics & Target Integrate

One of the most common questions we get: "What's in the AEP destinations list? Are CJA and Journey Optimizer there? What about Analytics and Target?" Let's clear up the confusion about how Adobe products integrate with Experience Platform.

The Quick Answer

Before we dive deep, here's the direct answer to the most common questions:

Product In Destinations List? How It Integrates
Customer Journey Analytics (CJA) No Data Consumer — reads AEP datasets directly
Journey Optimizer (AJO) No Native to AEP — shares same infrastructure
Adobe Analytics As Source Source Connector — sends data TO AEP
Adobe Target Yes Destination — receives segments for personalization
Adobe Campaign Yes Destination — receives audiences for campaigns

The reason for these differences? Each product has a fundamentally different relationship with AEP. Let's explore each one.

Understanding AEP's Architecture

To understand why some products are destinations and others aren't, you need to understand AEP's role in the Adobe ecosystem:

AEP is the Central Data Hub

Products integrate with AEP in four distinct ways:

  1. Sources — Send data INTO AEP (e.g., Adobe Analytics)
  2. Destinations — Receive data FROM AEP (e.g., Target, Campaign)
  3. Native Applications — Built on AEP infrastructure (e.g., Journey Optimizer)
  4. Data Consumers — Read AEP datasets for analysis (e.g., CJA)

Customer Journey Analytics (CJA)

CJA

Data Consumer

Is CJA in the AEP destinations list? No.

CJA doesn't appear as a destination because it uses a connection-based architecture. Rather than receiving activated audiences, CJA directly connects to and reads AEP datasets.

How CJA Uses AEP Data:

  • Connections: CJA creates connections to AEP datasets (both profile and event data)
  • Data Views: You define how CJA interprets the connected data
  • Analysis: CJA reads data directly from AEP's data lake for analysis
  • Audience Publishing: CJA can publish audiences BACK to AEP (reverse flow)

What CJA Contributes:

  • Cross-channel journey analysis across all AEP data
  • Flexible data modeling without schema changes
  • Ad-hoc exploration and reporting
  • Audience discovery that can be published to AEP
Key Point: CJA and AEP have a bidirectional relationship. CJA reads AEP datasets for analysis, and can publish discovered audiences back to AEP for activation. But it's not a "destination" in the traditional sense.

Journey Optimizer (AJO)

Journey Optimizer

Native to AEP

Is Journey Optimizer in the AEP destinations list? No.

Journey Optimizer is built natively on AEP. It doesn't need to be a "destination" because it directly accesses the same Real-Time Customer Profiles and segments that AEP uses.

How AJO Uses AEP Data:

  • Shared Profiles: AJO reads directly from AEP's Real-Time Customer Profile
  • Shared Segments: AEP segments are automatically available in AJO
  • Unified Identity: Uses the same identity graph as AEP
  • Event Triggers: AEP events can trigger AJO journeys in real-time

What AJO Contributes:

  • Journey orchestration across email, push, SMS, in-app
  • Real-time decisioning and offer management
  • Journey event data flows back into AEP
  • Message interaction data enriches profiles
Key Point: Think of Journey Optimizer as an application layer on top of AEP, not a separate product that receives data from AEP. When you create a segment in AEP, it's immediately available in AJO without any "activation" step.

Adobe Analytics

Adobe Analytics

Source

Is Adobe Analytics in the AEP destinations list? No — it's a Source.

Adobe Analytics is one of the primary data sources for AEP. The Analytics Source Connector streams data FROM Analytics TO AEP, not the other direction.

How Analytics Sends Data to AEP:

  • Analytics Source Connector: Backfills historical data and streams new data to AEP
  • Data Mapping: Analytics variables map to AEP XDM schema fields
  • Classification Data: Can include classification/lookup data
  • Report Suite Data: Choose which report suites to send

What Analytics Contributes to AEP:

  • Rich behavioral data from web and app
  • Historical interaction data
  • eVar and prop values mapped to profile attributes
  • Events that can trigger journeys or qualify for segments

Dual Collection Strategy:

Many organizations use Experience Platform Web SDK to send data to both Analytics AND AEP simultaneously:

  • Single tag implementation
  • Data flows to both products in parallel
  • Ensures data consistency
  • Future-proofs your implementation

Adobe Target

Adobe Target

Destination

Is Adobe Target in the AEP destinations list? Yes!

Target is a first-class destination in AEP. You can activate AEP segments to Target for personalization across web, mobile, and other channels.

How to Activate Segments to Target:

  1. Navigate to Destinations in AEP
  2. Find Adobe Target in the catalog
  3. Configure the destination connection
  4. Select segments to activate
  5. Map attributes if needed

What Target Does with AEP Data:

  • Same-page/Next-page Personalization: Edge-based segment sharing enables real-time decisions
  • Audience Targeting: Use AEP segments as Target audiences
  • Profile Attributes: Access unified profile attributes in Target activities
  • Recommendations: Use AEP data to power product recommendations

What Target Contributes Back:

  • A/B test and activity data can flow to AEP
  • Experience decisions enrich customer profiles
  • Conversion data closes the feedback loop
Important: For real-time personalization, ensure you're using the Edge Network integration. This enables same-page personalization decisions based on AEP segments without page reload latency.

Adobe Campaign

Adobe Campaign

Destination

Is Adobe Campaign in the AEP destinations list? Yes!

Both Adobe Campaign Standard and Campaign v8 are available as destinations in AEP. You can activate audiences for email, SMS, and direct mail campaigns.

Available Campaign Destinations:

  • Adobe Campaign Standard: Cloud-based campaign management
  • Adobe Campaign v8: Hybrid deployment option
  • Adobe Campaign Managed Services: Fully managed option

What Campaign Does with AEP Data:

  • Receives activated audiences for campaign targeting
  • Uses profile attributes for personalization
  • Triggers campaigns based on segment membership
  • Executes cross-channel campaigns (email, SMS, push, direct mail)

What Campaign Contributes Back:

  • Email opens, clicks, bounces flow back to AEP
  • Campaign engagement enriches profiles
  • Delivery and response data available for analysis

Use Cases: How They Work Together

Unified Customer View

Analytics → AEP → CJA
Analytics behavioral data flows to AEP, gets unified with other sources, and CJA provides cross-channel journey analysis.

Real-Time Personalization

AEP → Target
Segments built in AEP activate to Target for same-page personalization based on unified profile data.

Journey Orchestration

AEP → Journey Optimizer
AEP segments and events trigger personalized journeys across email, push, SMS, and in-app.

Campaign Targeting

AEP → Campaign
High-value segments identified in AEP activate to Campaign for targeted email and direct mail campaigns.

Example: Cross-Product Journey

Here's how data might flow through the ecosystem for a retail customer:

  1. Data Collection: Web SDK sends browsing data to both Analytics and AEP
  2. Profile Unification: AEP merges web behavior with CRM data, email engagement, and store purchases
  3. Segment Creation: AEP identifies "High-value customers who browsed but didn't purchase"
  4. Multi-Channel Activation:
    • Target: Shows personalized product recommendations on next visit
    • Journey Optimizer: Sends abandoned browse email sequence
    • Campaign: Includes in weekly promotional email
    • Google Ads: Retargets with display ads
  5. Analysis: CJA analyzes the cross-channel journey to measure effectiveness
  6. Feedback Loop: Engagement data flows back to AEP, updating profiles
