What is Adobe Experience Platform (AEP)?

Published February 20, 2025 · 15 min read

Adobe Experience Platform (AEP) represents Adobe's next-generation foundation for customer experience management. It serves as the central nervous system for collecting, unifying, and activating customer data across all channels and touchpoints. This guide provides a comprehensive overview of AEP's architecture, capabilities, and strategic value.

The Foundation of Modern Customer Experience

AEP addresses a fundamental challenge facing enterprises: fragmented customer data spread across dozens of systems, channels, and touchpoints. Traditional approaches create data silos that prevent organizations from understanding and serving customers effectively.

Adobe Experience Platform solves this by providing:

  • Unified Data Lake: Centralized storage for all customer data
  • Real-Time Customer Profiles: Unified view of each customer
  • Identity Resolution: Connecting data across devices and channels
  • AI/ML Services: Intelligent predictions and insights
  • Activation Destinations: Connect profiles to downstream systems

AEP Architecture Overview

Data Collection Layer

AEP provides multiple methods for ingesting data:

  • Web SDK (Alloy.js): JavaScript library for web data collection
  • Mobile SDK: Native SDKs for iOS and Android
  • Server API: HTTP APIs for server-side data
  • Source Connectors: Pre-built integrations with Adobe and third-party systems
  • Batch Ingestion: File-based data import (CSV, Parquet, JSON)

Data Lake

The Experience Data Lake stores all ingested data in a standardized format:

  • Petabyte-scale storage capacity
  • Schema-based data organization (XDM)
  • Support for structured and unstructured data
  • Query Service for SQL-based analysis

Real-Time Customer Profile

The Profile Store maintains unified customer views:

  • Real-time profile updates (milliseconds)
  • Identity graph for cross-device resolution
  • Computed attributes for derived values
  • Profile snapshots for historical analysis

Profile vs. Data Lake

The Data Lake stores raw events indefinitely for analysis. The Profile Store maintains current customer state for real-time activation. Both are essential but serve different purposes.

XDM: Experience Data Model

XDM is Adobe's standardized schema framework that ensures data consistency across all systems. Key concepts include:

Schema Classes

  • XDM Individual Profile: Person-level attributes (demographics, preferences)
  • XDM ExperienceEvent: Time-series behavioral data (page views, purchases)
  • Custom Classes: Organization-specific data structures

Field Groups

Reusable collections of fields for common data types:

  • Demographic Details
  • Personal Contact Details
  • Commerce Details
  • Web Details
  • Custom field groups for specific needs

Schema Example

{
  "$schema": "https://ns.adobe.com/xdm/schema/descriptor",
  "type": "object",
  "title": "Customer Profile",
  "description": "Unified customer profile schema",
  "properties": {
    "personalEmail": {
      "title": "Personal Email",
      "type": "object",
      "properties": {
        "address": {
          "type": "string",
          "format": "email"
        }
      }
    },
    "loyaltyTier": {
      "title": "Loyalty Tier",
      "type": "string",
      "enum": ["bronze", "silver", "gold", "platinum"]
    }
  }
}

Identity Resolution

Identity Service connects fragmented identities into unified profiles:

Identity Namespaces

  • ECID: Adobe's first-party device identifier
  • Email: Hashed email addresses
  • Phone: Hashed phone numbers
  • CRM ID: Customer system identifiers
  • Custom IDs: Organization-specific identifiers

Identity Graph

The identity graph links identities to create complete customer views:

  1. Anonymous visitor arrives (ECID assigned)
  2. Visitor logs in (Email linked to ECID)
  3. Mobile app installed (Device ID linked)
  4. In-store purchase (Loyalty ID linked)

Result: Four separate identities unified into one customer profile.

Real-Time CDP

Real-Time Customer Data Platform (CDP) is an application built on AEP that enables:

Audience Segmentation

  • Build segments using profile attributes and behaviors
  • Streaming evaluation for real-time membership
  • Scheduled evaluation for complex criteria
  • Look-alike modeling for audience expansion

Activation Destinations

Connect audiences to marketing channels:

  • Adobe Destinations: Target, Campaign, Journey Optimizer
  • Advertising: Google, Facebook, LinkedIn, The Trade Desk
  • Email: Salesforce Marketing Cloud, Marketo
  • Cloud Storage: S3, Azure, Google Cloud Storage
  • Custom: HTTP API for any destination

AI/ML Capabilities

Customer AI

Predictive models for customer behavior:

  • Propensity to convert
  • Propensity to churn
  • Lifetime value prediction

Attribution AI

Understand marketing channel effectiveness:

  • Algorithmic attribution modeling
  • Touchpoint analysis
  • Conversion path insights

Implementation Considerations

Data Governance

AEP includes robust governance capabilities:

  • Data labels for sensitive information
  • Usage policies to control activation
  • Consent management integration
  • Audit logging for compliance

Getting Started

  1. Define Use Cases: Identify specific business outcomes
  2. Data Inventory: Catalog existing data sources
  3. Schema Design: Create XDM schemas for your data
  4. Identity Strategy: Define identity namespaces and linking rules
  5. Source Integration: Connect data sources to AEP
  6. Segment Creation: Build audiences for activation
  7. Destination Setup: Connect to marketing channels

AEP vs. Other CDPs

Adobe Experience Platform differentiates from other CDPs through:

  • Real-Time Processing: Millisecond-level profile updates
  • Scale: Designed for enterprise data volumes
  • Adobe Integration: Native connection to Adobe Experience Cloud
  • Flexibility: Support for custom schemas and data models
  • AI/ML: Built-in intelligence services

Conclusion

Adobe Experience Platform provides the foundation for modern customer experience management. Its ability to unify data, resolve identities, and activate audiences in real-time enables organizations to deliver personalized experiences at scale. Success with AEP requires careful planning, proper data governance, and alignment with clear business objectives.
