AEM Cloud + Workfront Integration: Complete Setup Guide

Connect your creative project management with your digital asset management. This guide shows you how to integrate AEM as a Cloud Service with Adobe Workfront step by step.

Why Integrate Workfront with AEM?

Workfront handles project planning and creative workflows. AEM Assets manages your digital assets. When they work together, you get:

Linked Folders

Project folders in Workfront auto-create matching folders in AEM Assets.

Asset Sync

Files uploaded in Workfront automatically appear in AEM. No manual copying.

Metadata Flow

Project info from Workfront syncs to AEM asset metadata automatically.

Proof Integration

Use Workfront Proof for review cycles, then publish approved assets to AEM.

Integration Options

Adobe offers two ways to connect Workfront and AEM:

Recommendation

Most enterprises need the Enhanced Connector. The native integration only handles basic document linking. If you want linked folders and metadata sync, go with Enhanced.

Prerequisites

Before you start, make sure you have:

  • AEM as a Cloud Service with AEM Assets
  • Adobe Workfront license (Pro, Business, or Enterprise)
  • Enhanced Connector license (if using Enhanced Connector)
  • Admin access to both AEM and Workfront
  • Cloud Manager access for AEM

Step-by-Step: Enhanced Connector Setup

1 Install the Enhanced Connector

The connector installs via Cloud Manager:

  1. Go to Cloud Manager > Your Program > Environments
  2. Click the environment menu (three dots)
  3. Select "Add Enhancement"
  4. Find "Workfront for Experience Manager Enhanced Connector"
  5. Click Install and wait for deployment

2 Create Workfront API Key

In Workfront, generate an API key for AEM:

  1. Go to Setup > System > Customer Info
  2. Under API Keys, click "Generate API Key"
  3. Copy the key—you'll need it for AEM config
  4. Note your Workfront URL (e.g., yourcompany.my.workfront.com)

3 Configure AEM Cloud Service

In AEM Author, set up the Workfront connection:

  1. Go to Tools > Cloud Services > Workfront
  2. Click Create and choose a configuration folder
  3. Enter a title (e.g., "Workfront Production")
  4. Paste your Workfront instance URL
  5. Paste the API key
  6. Select a service user for the connection
  7. Click "Test Connection" to verify
  8. Save the configuration

4 Set Up Linked Folder Mapping

Define where Workfront folders appear in AEM:

  1. Open the Workfront cloud configuration
  2. Go to the "Linked Folders" tab
  3. Set the AEM base path (e.g., /content/dam/workfront)
  4. Configure folder naming (project name, portfolio, etc.)
  5. Set permissions for synced folders
  6. Save changes

5 Configure Metadata Mapping

Map Workfront fields to AEM metadata:

  1. Go to Tools > Assets > Metadata Schemas
  2. Edit your schema or create a new one
  3. Add fields for Workfront data (project name, campaign, etc.)
  4. In the Workfront config, go to "Metadata" tab
  5. Map Workfront custom form fields to AEM properties
  6. Choose sync direction (Workfront → AEM or bidirectional)

6 Enable in Workfront

Turn on the integration in Workfront:

  1. Go to Setup > Documents > Custom Integrations
  2. Enable "Experience Manager Assets"
  3. Enter your AEM Cloud Service URL
  4. Configure which users can access the integration
  5. Set document sync preferences

Using Linked Folders

Linked folders are the heart of the integration. When you create a project in Workfront, a matching folder appears in AEM Assets.

How It Works

  1. User creates a project in Workfront
  2. Enhanced Connector detects the new project
  3. AEM creates a folder in the configured location
  4. Folder inherits project metadata
  5. Documents added to Workfront sync to the AEM folder

Folder Structure Example

/content/dam/workfront/
├── 2025/
│   ├── Q1-Campaign/
│   │   ├── creative-briefs/
│   │   ├── final-assets/
│   │   └── working-files/
│   └── Product-Launch/
│       ├── photography/
│       └── video/
└── templates/
Pro Tip

Use folder templates in AEM to pre-create subfolders. When a linked folder syncs, it can auto-create "drafts", "approved", and "final" subfolders.

Metadata Mapping

Metadata mapping keeps asset information in sync. When you update project details in Workfront, the changes flow to AEM Assets.

Common Mappings

Workfront Field AEM Property Use Case
Project Name dc:title Asset titles
Portfolio dam:portfolio Brand grouping
Campaign Name dam:campaign Campaign tracking
Due Date dam:expirationDate Content expiry
Region dam:region Geo targeting
Status dam:status Approval state

Custom Form Sync

You can sync Workfront custom form fields to AEM. First, create matching properties in your AEM metadata schema. Then map them in the connector config: 

# Example metadata mapping configuration
workfront.metadata.mapping:
  - workfront: "DE:Product Line"
    aem: "dam:productLine"
    direction: "workfront-to-aem"
  
  - workfront: "DE:Usage Rights"
    aem: "dam:usageRights"
    direction: "bidirectional"
  
  - workfront: "DE:Expiration Date"
    aem: "dam:expirationDate"
    direction: "workfront-to-aem"

Proof Workflow Integration

If you use Workfront Proof, assets can flow through review before landing in AEM.

Typical Flow

  1. Designer uploads creative to Workfront
  2. Proof is created for stakeholder review
  3. Reviewers comment and approve/request changes
  4. Once approved, asset syncs to AEM Assets
  5. Asset is available for web publishing
Note on Versioning

When an asset updates in Workfront, a new version syncs to AEM. Configure your AEM versioning policy to avoid clutter. Consider auto-purging old versions.

Troubleshooting

Issue Solution
Connection test fails Check API key, verify Workfront URL, confirm firewall allows AEM IP
Folders not syncing Verify linked folder config, check service user permissions
Metadata not updating Confirm field mappings, check sync direction setting
Assets stuck in queue Check AEM replication queue, verify publish permissions
Duplicate folders Review folder naming rules, check for conflicting templates

Best Practices

  1. Plan your folder structure — Define naming conventions before enabling linked folders. Changes later are painful.
  2. Limit metadata mappings — Only sync fields you actually need. Too many mappings slow down sync and create clutter.
  3. Use folder templates — Consistent subfolders help teams find assets quickly.
  4. Set up alerts — Configure notifications for sync failures so you catch issues early.
  5. Train your team — Show creators where to upload files. Some should go to Workfront, others directly to AEM.
  6. Review permissions — Ensure the service account has correct access in both systems.

Conclusion

AEM + Workfront integration bridges the gap between creative production and digital asset management. The Enhanced Connector does the heavy lifting—syncing folders, assets, and metadata automatically.

Start with a pilot project. Test linked folders and metadata sync before rolling out company-wide. Once it's working, your creative teams will wonder how they ever worked without it.
