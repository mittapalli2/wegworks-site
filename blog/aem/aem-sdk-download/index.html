How to Download AEM SDK & Quickstart JAR: Complete Guide 2025

Published January 1, 2026 · 10 min read · AEM Development Guide

Setting up a local AEM development environment requires downloading either the AEM SDK (for Cloud Service) or the Quickstart JAR (for on-premise AEM 6.5). This guide walks you through the complete download process, including accessing Adobe Software Distribution, understanding version requirements, and initial setup steps.

Understanding AEM SDK vs Quickstart JAR

Before downloading, it's important to understand the difference between these two options:

AEM as a Cloud Service SDK

The AEM SDK is used for local development when building for AEM as a Cloud Service. It includes:

  • Quickstart JAR — The local AEM instance runtime
  • Dispatcher Tools — Local dispatcher for testing caching rules
  • Java API JAR — For compilation and dependency management

AEM 6.5 Quickstart JAR

For on-premise or AMS (Adobe Managed Services) deployments running AEM 6.5, you'll download:

  • Quickstart JAR — The main AEM runtime (e.g., aem-quickstart-6.5.x.jar)
  • License file — Required license.properties file
  • Service Packs — Optional but recommended updates

Which One Should You Download?

  • New projects → AEM Cloud Service SDK (recommended)
  • Existing AEM 6.5 projects → AEM 6.5 Quickstart JAR
  • Migration projects → Both (SDK for target, 6.5 for source)

How to Download AEM SDK (Cloud Service)

Follow these steps to download the AEM as a Cloud Service SDK:

Step 1: Access Adobe Software Distribution

  1. Go to Adobe Software Distribution
  2. Sign in with your Adobe ID (must have Cloud Service entitlement)
  3. Navigate to AEM as a Cloud Service section

Step 2: Select the SDK Version

In the downloads portal, look for "AEM SDK" entries. You'll see versions like:

  • aem-sdk-2024.11.18459.20241121T153621Z-241100.zip

⚠️ Version Matching

Always match your local SDK version to your Cloud Manager environment version. Check your Cloud Manager pipeline to see which SDK version is deployed.

Step 3: Download and Extract

  1. Click the download link for your required SDK version
  2. Extract the ZIP file to a local folder (e.g., ~/aem-sdk/)
  3. Inside, you'll find the Quickstart JAR and Dispatcher tools
# Extract the SDK
unzip aem-sdk-2024.11.18459.20241121T153621Z-241100.zip -d ~/aem-sdk/

# Navigate to the SDK folder
cd ~/aem-sdk/

# You'll see files like:
# - aem-sdk-quickstart-2024.11.18459.20241121T153621Z-241100.jar
# - aem-sdk-dispatcher-tools-2.0.xxx.zip

How to Download AEM 6.5 Quickstart JAR

For AEM 6.5 (on-premise or AMS), the download process is different:

Step 1: Access Adobe Licensing Website

  1. Go to Adobe Licensing Website
  2. Sign in with your organization's Adobe ID
  3. Navigate to your organization's entitlements

Step 2: Find AEM Downloads

  1. Select "Download Products"
  2. Search for "Experience Manager" or "AEM"
  3. Select version 6.5 and the latest Service Pack

Step 3: Download Required Files

Download these files:

  • AEM 6.5 Quickstart JAR — aem-quickstart-6.5.0.jar
  • License file — license.properties
  • Latest Service Pack — e.g., aem-service-pkg-6.5.21.zip

AEM SDK Version Compatibility

Understanding version compatibility is crucial for smooth development:

AEM Version Minimum Java Recommended Java Notes
AEM Cloud Service (2024+) Java 11 Java 11 or 17 Java 21 support coming
AEM 6.5.13+ Java 11 Java 11 Oracle or OpenJDK
AEM 6.5.0-6.5.12 Java 8 Java 11 Upgrade Java recommended

Setting Up Your Downloaded AEM

Directory Structure Best Practices

Organize your AEM installation properly:

# Recommended folder structure
~/aem/
├── author/
│   ├── aem-quickstart.jar
│   └── license.properties
├── publish/
│   ├── aem-quickstart.jar
│   └── license.properties
└── sdk/
    └── dispatcher-tools/

Renaming the JAR for Run Modes

AEM determines its run mode from the JAR filename:

# For Author instance (default port 4502)
mv aem-sdk-quickstart-xxx.jar aem-author-p4502.jar

# For Publish instance (default port 4503)
mv aem-sdk-quickstart-xxx.jar aem-publish-p4503.jar

# Custom port example
mv aem-sdk-quickstart-xxx.jar aem-author-p8080.jar

First-Time Startup

Run AEM for the first time:

# Navigate to your instance folder
cd ~/aem/author/

# Start AEM (first startup takes 5-10 minutes)
java -jar aem-author-p4502.jar

# Or with more memory allocated
java -Xmx4g -jar aem-author-p4502.jar

Troubleshooting Download Issues

Access Denied Errors

  • Problem: Cannot access Software Distribution
  • Solution: Ensure your Adobe ID is linked to an organization with AEM entitlements
  • Action: Contact your Adobe admin to grant download permissions

Slow Download Speeds

  • Problem: Download is extremely slow or times out
  • Solution: Use a download manager that supports resume
  • Alternative: Download during off-peak hours

Corrupted Downloads

  • Problem: JAR file won't start or shows corruption errors
  • Solution: Verify the file checksum if provided
  • Action: Re-download the file completely
