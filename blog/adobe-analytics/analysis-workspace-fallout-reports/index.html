Adobe Analytics Fallout Reports: Complete Analysis Workspace Tutorial

Published January 1, 2026 · 10 min read · Adobe Analytics Guide

Fallout reports in Adobe Analytics Analysis Workspace are essential for understanding conversion funnels and identifying where users drop off in their journey. This tutorial covers everything from basic fallout creation to advanced techniques for funnel optimization.

What is a Fallout Report?

A fallout report (also called funnel analysis) visualizes the conversion rate between sequential steps in a user journey. It shows:

  • Conversion Rate — Percentage of users completing each step
  • Drop-off Points — Where users abandon the journey
  • Funnel Health — Overall path efficiency

Creating a Basic Fallout Report

Step 1: Add Fallout Visualization

  1. Open Analysis Workspace
  2. In the left panel, find Visualizations
  3. Drag Fallout onto your panel
  4. A blank fallout visualization will appear

Step 2: Define Touchpoints

Touchpoints are the steps in your funnel. Add them by:

  1. Drag a dimension value (e.g., page name) to the "Add Touchpoint" dropzone
  2. Repeat for each step in your funnel
  3. Touchpoints can be pages, events, or dimension values

Touchpoint Types You Can Use

  • Pages — Specific page names or URLs
  • Events — Custom events (add to cart, video play)
  • Dimensions — Any dimension value
  • Segments — Complex criteria for touchpoints

Step 3: Configure Settings

Click the gear icon to access fallout settings:

Setting Options Use Case
Container Visitor / Visit / Hit Scope of the fallout analysis
Show all visits On / Off Include non-completing visits
Include repeats On / Off Count same step multiple times

Fallout Container Options

Visitor Container

Tracks fallout across multiple visits:

  • User sees homepage on Monday
  • User adds to cart on Wednesday
  • User purchases on Friday
  • Result: Counted as completing the funnel

Visit Container

Tracks fallout within a single session:

  • All steps must occur in one visit
  • More strict measurement
  • Better for immediate conversion paths

Hit Container

Tracks fallout within sequential hits:

  • Steps must be consecutive page views
  • Most strict measurement
  • Good for form completion analysis

⚠️ Container Impact on Results

Changing the container will significantly impact your numbers. Visitor containers typically show higher conversion rates than Visit or Hit containers. Choose based on your analysis goals.

Advanced Fallout Techniques

1. Segment Comparison

Compare fallout between different user segments:

  1. Create your fallout visualization
  2. Drag a segment onto the fallout panel
  3. Add another segment to compare
  4. View side-by-side conversion rates

2. Fallout After Touchpoint

Analyze where users go after dropping off:

  1. Right-click on a touchpoint
  2. Select "Fallout after this touchpoint"
  3. See top pages/actions after drop-off

3. Using Segments as Touchpoints

Create complex touchpoint criteria:

  • Users who viewed 3+ products
  • Users from paid search who signed up
  • High-value customers who made repeat purchases

4. Combining with Flow Visualization

Use Flow to understand paths to/from touchpoints:

  1. Right-click on any touchpoint
  2. Select "Create Flow from this touchpoint"
  3. Analyze entry/exit paths

  4. Form Complete
  5. Thank You Page

  1. Article Page View
  2. Scroll to 50%

  5. Newsletter Signup

Optimizing Based on Fallout Data

Identifying Problem Areas

  • High drop-off at checkout: Check for UX issues, form complexity
  • Product to cart drop-off: Review pricing, product info quality
  • Cart abandonment: Analyze shipping costs, payment options

A/B Testing Opportunities

Use fallout insights to guide testing:

  • Test simplified checkout flows
  • Test different CTAs at drop-off points
  • Test form field reduction
